# Техническое задание
## Проект: «Анализ активности open-source репозиториев»

---

## 1. Общая информация

| Параметр | Значение |
|----------|----------|
| **Название проекта** | Сравнительный анализ активности open-source репозиториев ML-фреймворков |
| **Объекты анализа** | PyTorch, TensorFlow, Keras |
| **Источник данных** | GitHub API |
| **Инструмент визуализации** | Yandex DataLens |
| **Период анализа** | Последние 52 недели (1 год) |

---

## 2. Цели и задачи проекта

### 2.1. Цель

Провести сравнительный анализ активности трёх ведущих ML-фреймворков (PyTorch, TensorFlow, Keras) на основе метрик GitHub и визуализировать результаты для принятия выводов о состоянии и развитии проектов.

### 2.2. Задачи

1. Сбор данных из GitHub API по всем трём репозиториям
2. Проектирование и наполнение базы данных
3. Построение дашбордов в Yandex DataLens
4. Подготовка аналитического отчёта
5. Создание презентации результатов

## 4. Перечень собираемых данных

### 4.1. Общие метрики репозитория

#### Основное:
- `repo_name` — название репозитория
- `stars_count` — количество звёзд 
- `forks_count` — количество форков
- - `updated_at` — дата последнего обновления
- - `created_at` — дата создания репозитория
#### Вторично:
- `language` — основной язык
- `watchers_count` — количество наблюдателей 
- `open_issues_count` — количество открытых issues (из API)

### 4.2. Коммиты (weekly commits за 52 недели)

- `repo_name` — название репозитория
- `week_start_date` — дата начала недели (timestamp → дата)
- `commits_count` — количество коммитов за неделю (в среднем)

### 4.3. Issues

- `repo_name` — название репозитория
- `issue_state` — состояние (open / closed)
- `created_at` — дата создания
- `closed_at` — дата закрытия (если closed)
- `period` — нумерация по месяцам/неделям

### 4.4. Pull Requests

- `repo_name` — название репозитория
- `pr_state` — состояние (open / closed / merged)
- `created_at` — дата создания
- `merged_at` — дата слияния (если merged)

### 4.5. Контрибьюторы

- `repo_name` — название репозитория
- `contributor_login` — логин пользователя
- `contributions_count` — количество вкладов

### 4.6. Релизы

- `repo_name` — название репозитория
- `release_tag` — тег релиза (версия)
- `release_name` — название релиза
- `published_at` — дата публикации
- `is_prerelease` — флаг пре-релиза

## 5. Схема базы данных

### Денормализованная схема (для Yandex DataLens)
#### Таблица `repo_summary.csv`

| Поле | Тип | Описание |
|------|-----|----------|
| repo_name | STRING | Название репозитория |
| stars | INT | Количество звёзд |
| forks | INT | Количество форков |
| total_commits_52w | INT | Всего коммитов за 52 недели |
| total_contributors | INT | Всего контрибьюторов |
| total_releases | INT | Всего релизов |
| last_release_date | DATE | Дата последнего релиза |
| days_since_last_release | INT | Дней с последнего релиза |
| open_issues | INT | Открытые issues |
| closed_issues_52w | INT | Закрытые issues за год |

#### Таблица `weekly_activity.csv`

| Поле | Тип | Описание |
|------|-----|----------|
| repo_name | STRING | Название репозитория |
| week_date | DATE | Дата начала недели |
| commits | INT | Коммиты за неделю |
| quarter | STRING | Квартал (Q1-Q4 2024/2025) |

#### Таблица `contributors.csv`

| Поле | Тип | Описание |
|------|-----|----------|
| repo_name | STRING | Название репозитория |
| contributor_login | STRING | Логин контрибьютора |
| contributions | INT | Количество contributions |

#### Таблица `releases.csv`

| Поле | Тип | Описание |
|------|-----|----------|
| repo_name | STRING | Название репозитория |
| release_tag | STRING | Тег версии |
| published_at | DATETIME | Дата публикации |
| is_prerelease | BOOLEAN | Пре-релиз или нет |

## 6. Источники данных (GitHub API endpoints)

| Данные | API Endpoint | Примечание |
|--------|--------------|------------|
| Общая информация | `GET /repos/{owner}/{repo}` | stars, forks, open_issues |
| Weekly commits | `GET /repos/{owner}/{repo}/stats/participation` | Массив за 52 недели |
| Contributors | `GET /repos/{owner}/{repo}/contributors` | Пагинация, до 100 на страницу |
| Releases | `GET /repos/{owner}/{repo}/releases` | Пагинация |
| Issues | `GET /repos/{owner}/{repo}/issues?state=all` | Фильтр по дате, пагинация |
| Pull Requests | `GET /repos/{owner}/{repo}/pulls?state=all` | Пагинация |

**Репозитории:**

- `pytorch/pytorch`
- `tensorflow/tensorflow`
- `keras-team/keras`

- Timeline релизов 

#### Таблицы

- Сводная таблица метрик по репозиториям
- Топ контрибьюторов каждого репозитория
- Список релизов с датами
