# Boolean Algebra и функции

### Boolean function & Boolean formula

- **Функция**: отображение `{0,1}ⁿ → {0,1}`
    
- **Формула**: способ записи функции через AND, OR, NOT
    

### Number of n-ary Boolean functions

- Всего `2^(2^n)` функций из n переменных
    
- Пример: 2 переменные → `2^(2^2) = 16` функций
    

---

# Основные элементы

### Minterm & Maxterm

- **Minterm**: AND всех переменных/отрицаний → ровно 1 набор делает = 1
    
- **Maxterm**: OR всех переменных/отрицаний → ровно 1 набор делает = 0
    

### DNF & CNF

- **DNF**: OR минтермов (сумма произведений)
    
- **CNF**: AND макстермов (произведение сумм)
    

### Negation normal form (NNF)

- NOT применяется только к переменным, не к целым формулам

# Расширенные формы и разложения
## Shannon expansion (по-человечески)

**Идея:**

> Берём переменную x и рассматриваем два случая: x = 0 и x = 1.  
> Функцию f можно “разложить” на две подфункции — одну для x=1, другую для x=0.

---

### Пример на пальцах

Функция:

```
f(x, y) = x AND y
```

- Если x = 1 → f = y
    
- Если x = 0 → f = 0
    
### Краткая фраза для коллоквиума

> “Shannon expansion — это способ разбить функцию по одной переменной на два случая, чтобы упростить её анализ и минимизацию.”

$\space$
### Algebraic normal form (ANF / Zhegalkin)

- XOR + AND + константа 1
    
- Пример: f = x ⊕ y ⊕ xy
    
- **Методы построения:** по таблице истинности, по минтермам, по рекурсии
    

### Gray code

- Порядок комбинаций для минимизации функции, меняется только 1 бит

# Кубы и импликанты

### Literal, clause, cube

- **Literal:** x или ¬x
    
- **Clause:** OR литералов
    
- **Cube:** AND литералов (как минтерм)
    

### Implicant, prime implicant, essential prime implicant

- **Implicant:** куб, покрывающий часть единиц функции
    
- **Prime implicant:** нельзя упростить, чтобы он по-прежнему покрывал 1
    
- **Essential prime implicant:** покрывает единицу, которую не покрывает никто другой

# Минимизация функций

### Karnaugh map (K-map)

- Таблица или схема, где единицы соединяем в группы (1,2,4,8…)
    
- **Don't-care conditions:** значения, которые можно использовать для упрощения
    

### Quine-McCluskey algorithm

- Табличный метод минимизации через:
    
    1. Сортировку минтермов по числу единиц
        
    2. Постепенное объединение в большие кубы
        
    3. Выделение prime implicants
        

### Petrick's method (exact cover)

- Если остаются несколько вариантов покрытия единиц → формируем выражение для **выбора минимального набора импликантов**
    
- Работает как “система уравнений” для минимального покрытия

# Функциональный анализ

### Superposition (composition) & functional closure

- **Superposition:** подстановка функций в функции
    
- **Functional closure:** множество всех функций, которые можно построить из данного набора
    

### Functional completeness

- Множество функций **полно**, если можно выразить любую булеву функцию
    

### Post's criterion & classes (T0, T1, S, M, L)

- **T0:** сохраняет 0
    
- **T1:** сохраняет 1
    
- **S:** самодвойственные
    
- **M:** монотонные
    
- **L:** линейные
    
- **Полнота:** множество **не принадлежит ни одному из этих классов полностью**
    

### Sheffer stroke & Peirce arrow

- **NAND (Sheffer)** и **NOR (Peirce)**
    
- Любую булеву функцию можно выразить через одну из этих операций
